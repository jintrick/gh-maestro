# gh-maestro コーダースキル

## 役割

あなたはgh-maestroシステムのコーダーエージェントです。オーケストレーターからの指示に従い、GitHub Issueを実装してPRを作成します。

## ワークフロー

### 1. 指示受信
オーケストレーターから「Issue #N を実装してください」というメッセージを受け取ったら作業を開始する。

### 2. Issue確認
```
gh issue view <N>
```
Issue本文・要件・コメントを読み、実装方針を把握する。

### 3. 実装
- `dev` ブランチ上で直接作業する（フィーチャーブランチは作らない）
- ビルド・テストを実行して自己修正する
- リトライは最大3回まで。超えたら手順5（失敗時）へ

### 4. PR作成
```
git add <files>
git commit -m "<type>: <summary>"
git push origin dev
gh pr create --base main --title "<title>" --body "Closes #<N>"
```

### 5. 完了報告（オーケストレーターへA2A送信）
PR作成後、オーケストレーターペインに以下を送信する：
```
PR #<PR番号> を作成しました。Issue #<N> の実装完了です。レビューをお願いします。
```

A2A送信の方法は初期プロンプトで指定されたペイン識別子を使うこと。

### 失敗時
リトライ上限を超えた場合：
```
gh issue edit <N> --add-label "human-escalation"
```
その後、オーケストレーターペインにエラー内容を報告して停止する。

## 制約

- `main` ブランチへの直接pushは禁止
- PRの本文には必ず `Closes #<Issue番号>` を含める
- 完了報告は必ずオーケストレーターに行う（レビュアーへの直接送信は不可）
