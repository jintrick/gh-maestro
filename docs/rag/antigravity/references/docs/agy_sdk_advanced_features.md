# Antigravity SDK: Advanced Capabilities

The SDK provides several features for building professional-grade, multimodal, and cooperative agent systems.

## 1. High-Level Streaming
Access live reasoning and output chunks as they are generated by the model.

```python
async with Agent(config) as agent:
    response = await agent.chat("Write a long story.")
    
    # Stream text tokens
    async for token in response:
        print(token, end="", flush=True)
        
    # Stream reasoning "thoughts"
    async for thought in response.thoughts:
        update_thinking_ui(thought)
```

## 2. Multimodal Input
Pass images, PDFs, audio, and video natively using semantic content classes.

```python
from google.antigravity.types import Image, Document

async with Agent(config) as agent:
    # Pass a local screenshot for analysis
    img = Image.from_file("screenshot.png")
    await agent.chat(["Explain this UI", img])
```

## 3. Structured Output (Pydantic)
Define schemas to ensure the agent returns validated, typed Python objects.

```python
class Task(Pydantic.BaseModel):
    title: str
    priority: int

async with Agent(config) as agent:
    result = await agent.chat("Create a task", response_format=Task)
    print(result.title) # result is a Task instance
```

## 4. Human-in-the-Loop
The SDK handles blocking questions from the agent automatically via the `InteractionHook`.
- **AskUserQuestion**: The agent pauses and waits for structured user input.
- **Branching**: The conversation trajectory forks based on the user's choice.

## 5. Sub-agents & Multi-Agent Teams
Agents can spawn child agents with specialized toolsets and independent contexts.
- **`START_SUBAGENT` Tool**: Used internally when an agent delegates tasks.
- **Inherited Scopes**: Sub-agents automatically inherit the parent's directory permissions and terminal restrictions, but have their own thinking loop.

## 6. Observability
- **Token Tracking**: Per-turn and cumulative token counters.
- **Thinking Traces**: Access the model's internal chain-of-thought (CoT) even if not streamed.
- **Harness Logs**: Access detailed logs from the underlying Go harness via `Connection`.
